# Roadmap

Pending features + known gaps. Completed work lives in git history, not here.

## Gaps to close

- [ ] **Security screen** — 2FA toggle and "cerrar todas las sesiones" are no-ops; biometric state is local-only. Wire to real backend state (Supabase Auth factors + session revocation) when the feature is prioritized.
- [ ] **Support screen** — email / teléfonos / horario are hardcoded in the screen. Move to a `support_contacts` table (or env-driven config) so marketing can update without a release.
- [ ] **Legal text screen** — `LegalContent.terms` + `LegalContent.privacy` live as `static const` strings. Move to a `legal_documents` table (versionable, with `effective_date`) so a new version can be published + tracked without rebuilding the app.
- [ ] **Search trending tags** — the idle-state tag list is a `const` in `search_screen.dart`. Ideal source: analytics-driven (most-searched over the last N days) or admin-managed. Low priority until we have data volume.
- [ ] **Real floor-plan images** — the bundle mock was removed. 11 projects currently have `floor_plan_url = NULL` and the PLANO section is hidden for those. Upload real floor plans to Supabase Storage when material is available (no code change needed — the UI already handles the NULL branch).
- [ ] **Forgot password flow** — login screen has the link but the target flow is not implemented. Supabase Auth supports the reset email out of the box — needs a screen + deep link handler.
- [ ] **Welcome screen video** — currently a stock Coverr clip. Replace with branded content before production. Infrastructure (signed-URL system, `LhotseVideoPlayer`, `FullscreenVideoPlayer`) is already in place — only the MP4 asset swap is needed.
- [ ] **Asset detail screen** — `FeedAssetItem` (Home feed) tap is currently a no-op. Decide whether it links to a standalone asset screen, an equivalent of project detail, or a bottom sheet. Hero tag `asset-hero-{id}` is already in place so the shared-element transition lands when a destination exists.
- [ ] **NAV (Net Asset Value) hero migration** — today the hero `totalAmount` (L1 patrimonio + L2 by brand) is **active capital invested** (sum of `purchaseValue` / `amount` over `!isCompleted`). NAV (capital × current valuation) is the wealth-banking standard but requires: (a) current-valuation tracking per coinv contract (no field today), (b) consistent treatment of RF (face value vs market value — no market data tracked), (c) decision on whether to include realized gain from completed contracts in lifetime-AUM views (separate from active NAV). This is a coordinated change across L1 + L2 × 3 business models, not a card-level refactor. Deferred until pre-requisites land.
- [ ] **Compra Directa rental received tracking** — `PurchaseContractData` exposes `monthlyRent` and `rentalYieldPct` (forward-looking) but no `rentalReceivedToDate` field. Without it, the Compra Directa active card cannot show the `Recibido +€` accumulated gain that RF surfaces — a parity gap. Adding the field requires either (a) schema column with manual updates per month, or (b) computed view summing `rent_payments` table (if/when one exists). Once available, extend `_PurchaseRow` meta to mirror RF structure: `{yield}% anual · Recibido +€`.
- [ ] **Wire up document tap → preview with Supabase Storage** — `LhotseDocRow.onTap` is currently no-op across the strategy detail screens (CD active, CD/coinv completed, coinv active) and the search screen. The L3 row visually indicates tap-affordance via the chevron, but tapping does nothing. Implementation per `docs/sql/MIGRATION_CHECKLIST.md` etc.: (1) extend `DocumentData → LhotseDocument` mapping to carry `fileUrl` (currently dropped). (2) Add `open_filex` and `path_provider` dependencies. (3) Helper `lib/core/utils/open_supabase_doc.dart` that generates a signed URL (`supabase.storage.from(bucket).createSignedUrl(path, 60)`), downloads to `getTemporaryDirectory()` with cache by `doc.id`, opens via `OpenFilex.open(localPath)`. iOS gets Quick Look + native share sheet, Android gets system viewer (Drive/Adobe) with their built-in share. (4) Loading state: replace chevron with 16pt CircularProgressIndicator while downloading. (5) Error: SnackBar with retry. Confirm with backend whether `documents.file_url` already arrives as a signed/public URL or a storage path requiring signing client-side.
